1. Strategic Context

Closo.co reached ~130K monthly organic traffic in under 6 months publishing 50+ reseller articles. Every single article is generic — zero live data, zero inventory proof, zero real margin numbers. They manufacture authority from nothing.

888lots has the inverse problem. Real GSC traffic. Real inventory. Real margins. Real seasonality from years of order history. Almost no content converting that data into organic search presence. The gap is purely execution.

Core thesis: Closo writes about the market. 888lots IS the market. Every article this pipeline produces can contain data Closo structurally cannot replicate — live GSC signals, actual BSR-ranked products, real margin percentages, order-level seasonality. That is the moat.

The Closo Formula — Decoded

50 articles. Three title formulas used across all content. All end in the current year for SERP freshness. Tone: 'Honest / Brutal / Reality Check' — signals E-E-A-T trust to Google. Topics are 100% educational, zero proprietary data.

Type 2 — Brand Spotlight

What it is: One article per brand, fires when on_stock transitions from 0 to positive. Targets '[Brand] wholesale' searches — highest commercial intent of all four types. These convert browsers into buyers.

SELECT id, slug, title, platform, url,
  on_stock, total_stock, last_week_load,
  avg_profit_margin, avg_net_profit,
  top_products_bsr, typical_product_keywords,
  safe_b2b_keywords, units_moved_week,

ORDER BY (on_stock + COALESCE(units_moved_week,0)) DESC NULLS LAST;

Trigger Logic

  • Supabase webhook fires on UPDATE to n8n_canonical_pages WHERE page_type = 'brand'
  • N8N IF check: new on_stock > 0 AND old on_stock = 0 → generate new article
  • Existing article < 30 days old → PATCH WP post with fresh data, no new post
  • Brand goes out of stock → PATCH existing post: add 'currently unavailable' CTA
  • Existing article >= 30 days + brand restocked → regenerate fresh article

Type 4 — Seasonal Alert

What it is: Fires 3–4 weeks before a category's peak demand month — based on month_scores derived from actual order history. Only triggers when seasonality_confidence is medium or high. Gives resellers the lead time they need.

Title Formula:
"[Category] Season Starts [Month]: Why Smart Resellers Are Sourcing Now — 888lots Data"

Real examples that trigger from current canonical data:

  • Fragrance Season Starts December: 4.1x Demand Spike Confirmed — Source Now at 888lots (fires October)
  • Sneakers Peak Window: July is Your Best Month — 888lots Order Data (fires May/June)
  • Handbags Peak: May–June Confirmed by 3-Year Order History — Start Sourcing March (fires March)
SELECT id, slug, title, platform,
  month_scores, seasonality_confidence, buyer_insight,
  avg_profit_margin, forecast_next_month,
  safe_b2b_keywords, gsc_impressions_current,
  on_stock, stock_urgency, url, last_data_update
FROM n8n_canonical_pages
WHERE page_type = 'category'
  AND seasonality_confidence IN ('medium', 'high')
  AND month_scores IS NOT NULL
  AND (month_scores->>to_char(NOW() + INTERVAL '5 weeks', 'Mon'))::int
    > (month_scores->>to_char(NOW(), 'Mon'))::int * 1.25
  AND id NOT IN (
    SELECT canonical_page_id FROM n8n_blog_articles
    WHERE article_type = 'seasonal_alert'
    AND generated_at > NOW() - INTERVAL '10 months'
  );

4. Infrastructure — n8n_blog_articles Table

All four workflows write to this single table. It is the system memory — without it, workflows regenerate duplicates, WordPress accumulates redundant drafts, and reporting is impossible. Create this before building any workflow.

CREATE TABLE n8n_blog_articles (
  id                    SERIAL PRIMARY KEY,
  canonical_page_id     INT REFERENCES n8n_canonical_pages(id),
  platform              TEXT NOT NULL,
  article_type          TEXT NOT NULL,
    -- 'category_guide' | 'brand_spotlight' | 'weekly_pulse' | 'seasonal_alert'
  title                 TEXT,
  primary_keyword       TEXT,
  html_body             TEXT,
  word_count            INT,
  wp_post_id            INT,
  wp_status             TEXT DEFAULT 'draft',
    -- 'draft' | 'publish' | 'update_queued' | 'skipped'
  quality_gate_passed   BOOLEAN DEFAULT FALSE,
  generated_at          TIMESTAMPTZ DEFAULT NOW(),
  published_at          TIMESTAMPTZ,
  gsc_clicks_at_publish INT,
  impressions_at_publish INT,
  data_snapshot         JSONB,
    -- full canonical row frozen at generation time — audit trail
  error_log             TEXT,
  refresh_after         DATE
    -- NULL for pulse/spotlight. NOW()+1yr for category guides.
);

CREATE INDEX idx_blog_canonical  ON n8n_blog_articles(canonical_page_id);
CREATE INDEX idx_blog_type_plat  ON n8n_blog_articles(article_type, platform);
CREATE INDEX idx_blog_wp         ON n8n_blog_articles(wp_post_id);
CREATE INDEX idx_blog_refresh    ON n8n_blog_articles(refresh_after)
  WHERE refresh_after IS NOT NULL;

Deduplication Check — Every Workflow Runs This First

SELECT id, wp_post_id, generated_at, quality_gate_passed
FROM n8n_blog_articles
WHERE canonical_page_id = {{canonical_id}}
  AND article_type = '{{article_type}}'
  AND (refresh_after IS NULL OR refresh_after > NOW())
LIMIT 1;

-- Row exists + refresh_after > NOW()  →  SKIP
-- Row exists + refresh_after <= NOW() →  REGENERATE, PATCH existing WP post
-- No row found                        →  GENERATE fresh, INSERT new WP post

7. Build Sequence

Principle: Build in order of value × complexity. Category Guides deliver the most SEO value and use the simplest trigger (manual batch). Prove the pipeline end-to-end on them first before adding webhook and cron complexity.

Week 1 — Foundation

  1. Step: Create n8n_blog_articles table in Supabase with all columns and indexes.
  2. Set up WordPress Application Password. Store as N8N credential 'wp_888lots'.
  3. Test WP REST API manually: POST one draft, confirm category IDs and tag creation work.
  4. Build WF-01 nodes 1–9 in order. Do not skip error logging node.
  5. Run WF-01 on top 5 categories by gsc_impressions_current. Review 5 WP drafts.
  6. Check quality: does each article cite real data? Does word count hit target? Is HTML clean?
  7. Adjust system prompt if tone or structure is off. Re-run. Approve batch.

Week 2 — Scale Category Guides + Brand Spotlight

  1. Step: Run WF-01 on all 100 888lots categories. Expect ~85 to pass quality gate.
  2. Editor spot-checks 10% (8–10 articles). Flag any structural issues.
  3. Build WF-02 webhook trigger. Test: manually update on_stock for 1 brand, confirm workflow fires.
  4. Run WF-02 on all currently in-stock brands (888lots: 56, miamilots: 477).
  5. Review 10 Brand Spotlight drafts. Approve batch.

Week 3 — Weekly Pulse + Seasonal Alert

  1. Step: Build WF-03 Weekly Pulse. Test with current week's live data.
  2. Verify quality gate correctly blocks thin weeks (manually zero rows in test environment).
  3. Build WF-04 Seasonal Alert. Test trigger logic with mock month_scores.
  4. Run WF-04 — identify which categories currently qualify. Review outputs.
  5. Set Monday 06:00 UTC cron for WF-03 across all 5 platforms.
  6. Set daily 08:00 UTC cron for WF-04 across all 5 platforms.

Month 2+ — Steady State

  • WF-01: monthly refresh pass — regenerate articles where refresh_after <= NOW()
  • WF-02: continuous fire as brands come into stock — zero manual intervention
  • WF-03: fires every Monday — editor reviews Tuesday morning — publishes same day
  • WF-04: fires on seasonal triggers — editor reviews within 48 hours
  • Expand to miamilots: re-run WF-01 batch (Spanish prompt variant for LATAM audience)
  • Expand to britdeals: re-run on britdeals categories (UK English + GBP pricing)
